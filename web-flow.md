# 🌐 網站開發流程（含負責人、優先級與可平行任務）

---

## 🧭 規劃階段

| 任務                      | 負責人           | 優先級  | 可否平行                  |
| ------------------------- | ---------------- | ------- | ------------------------- |
| 專案目標定義與需求訪談    | PM / 客戶 / 前端 | ✅ 必要 | ❌ 不可平行               |
| 功能需求文件撰寫 (PRD)    | PM               | ✅ 必要 | ❌ 不可平行               |
| 使用者流程圖（User Flow） | UX 設計師        | ✅ 必要 | ✅ 可與線框圖平行         |
| 資訊架構（IA）設計        | UX 設計師        | ✅ 必要 | ✅ 可平行                 |
| 頁面線框圖（Wireframe）   | UX / UI 設計師   | ✅ 必要 | ✅ 可與設計稿平行進行部分 |
| 技術棧決策（Tech Stack）  | 前端 / 後端 Lead | ✅ 必要 | ❌ 不可平行               |

---

## 🎨 設計階段

| 任務                          | 負責人             | 優先級    | 可否平行                |
| ----------------------------- | ------------------ | --------- | ----------------------- |
| 設計系統建立（Design System） | UI 設計師          | ✅ 必要   | ✅ 可與頁面設計平行     |
| 頁面視覺稿（Hi-Fi Mockup）    | UI 設計師          | ✅ 必要   | ✅ 可平行於部分前端開發 |
| 動態互動稿（Prototype）       | UI / 前端          | ⚪ 非必要 | ✅ 可平行               |
| 設計審核（Design Review）     | PM / 設計師 / 前端 | ✅ 必要   | ❌ 不可平行             |

---

## 💻 前端開發階段

| 任務                                | 負責人      | 優先級    | 可否平行              |
| ----------------------------------- | ----------- | --------- | --------------------- |
| 前端框架初始化（Vite / Next.js 等） | 前端工程師  | ✅ 必要   | ✅ 可與設計完成後平行 |
| UI 元件開發（Storybook）            | 前端工程師  | ✅ 必要   | ✅ 可平行             |
| API 介接規格文件確認                | 前端 / 後端 | ✅ 必要   | ❌ 不可平行           |
| 頁面功能開發（含狀態管理）          | 前端工程師  | ✅ 必要   | ✅ 可平行於後端開發   |
| 動畫與互動優化                      | 前端工程師  | ⚪ 非必要 | ✅ 可平行             |
| RWD / 跨裝置測試                    | 前端工程師  | ✅ 必要   | ✅ 可平行             |

---

## ⚙️ 後端開發階段

| 任務                                      | 負責人              | 優先級    | 可否平行            |
| ----------------------------------------- | ------------------- | --------- | ------------------- |
| 資料庫設計（Schema）                      | 後端工程師          | ✅ 必要   | ❌ 不可平行         |
| API 規格定義（OpenAPI / Swagger）         | 後端工程師          | ✅ 必要   | ✅ 可平行於前端準備 |
| 認證與權限機制                            | 後端工程師          | ✅ 必要   | ✅ 可平行           |
| 第三方服務整合（Webhook、Payment、OAuth） | 後端工程師          | ⚪ 非必要 | ✅ 可平行           |
| 單元測試（Unit Test）                     | 後端工程師          | ✅ 必要   | ✅ 可平行           |
| API 部署與版本控管                        | 後端工程師 / DevOps | ✅ 必要   | ✅ 可平行           |

---

## 🧩 標註總覽

| 標註        | 意義                             |
| ----------- | -------------------------------- |
| ✅ 必要     | 專案核心任務，缺少將影響產品交付 |
| ⚪ 非必要   | 可視情況調整或延後執行           |
| ✅ 可平行   | 可與其他任務同時進行             |
| ❌ 不可平行 | 須等前一階段完成再進行           |
